# Glossary

This glossary defines key terms and concepts used throughout the OSC system and documentation.

## A

**API** - Application Programming Interface. A set of protocols for building software applications.

**Authentication** - The process of verifying the identity of a user or system.

## D

**Dashboard** - The main control panel where users can view metrics and access features.

**Database** - Organized collection of structured data stored electronically.

## E

**Endpoint** - A specific URL that represents a service or resource.

**Environment** - A configured system setup (development, staging, production).

## O

**OSC** - One Stop Centre. The main system covered in this documentation.

## P

**Production** - The live environment serving real users.

**Protocol** - A set of rules governing data transmission.

## S

**Staging** - Pre-production environment for testing.

**System** - The complete OSC application and infrastructure.
